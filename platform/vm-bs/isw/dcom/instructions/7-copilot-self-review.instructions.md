# Instruction 6: Copilot Self-Review Checklist result(.md) file

For every output file generated as part of Detailed Software Design (DSD), Unit Test (UT), or Implementation activities (as per all instructions and prompts):

- After generating any output file, update a single, unified self-review checklist result file under the `DCOM_DID/review/` folder named `COPILOT_SELF_REVIEW_CHECKLIST_RESULT.md`.
- In this result file, create a separate section for each output file (DSD, UT, implementation, etc.).
- For each section, provide:
  - The Copilot Self-Review Checklist, marking each item as satisfied (), not satisfied (), or with comments explaining how it was addressed or why it was not satisfied for that specific output.
  - A concise summary of the generated file, describing its purpose, main contents, and how it fulfills the requirements or design intent.
- For every output file generated, log all review points, feedback, and actions taken during the review process. This ensures full traceability and documentation of the review process for each deliverable.
- This process must be followed for every DSD, UT, and implementation deliverable, ensuring traceable and auditable compliance with all project instructions and prompts in a single, consolidated file.
